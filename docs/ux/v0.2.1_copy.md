# v0.2.1 UI Copy Sheet — People First (LOCKED scope)

**Last Updated:** 2026-02-13  
**Owner:** UX Writer (Product Clarity)  
**Scope:** Copy + labels + tooltips + confirmations for *Household Card UI* and *life logs* (no new mechanics).

---

## Global voice + formatting rules

- **Status-oriented.** Put the outcome first, then the reason (if any).
- **Short.** One sentence per line in banners/cards.
- **Names before roles** on first mention: “{name} — {role}”.
- **Numbers with units**: “Food: 312 bushels”, “Unrest: 47/100”.
- **Avoid lore/flavor** in v0.2.1. Clarity > vibe.

### Locked terms (do not rename)
- **Dispossessed**
- **DeathNoHeir**
- **Heir**
- **Widow / Widower**

## Household Card (Turn Report) — REQUIRED

### Section title
- **Household**

### Card header line (primary identity)
- **{ruler_name} — {title}**  
  Secondary: **Age {age}** *(omit if age unavailable)*

If ruler name missing:
- **Unnamed ruler** *(fallback; should be rare)*

### Badges (short, single-purpose)
- **Heir** *(on the current heir)*
- **Widow** / **Widower** *(on the surviving spouse; only when applicable)*

Tooltip:
- **Heir** — Next valid successor under the realm’s rules.
- **Widow/Widower** — Surviving spouse of the prior ruler.

### Key rows (compact)
- **Heir:** {heir_name} *(or “Heir: None”)*  
- **Spouse:** {spouse_name} *(or “Spouse: None”)*  
- **Children:** {children_count}

Children compact list (optional, if space allows):
- **Children:** {child_1_name} (Age {x}), {child_2_name} (Age {y}) … **+{n_more}**

If children_count = 0:
- **Children:** None

### Last succession note (compact, 1 line)
- **Last succession:** Turn {turn_index} — {new_ruler_name} assumes rule.

If no succession has occurred in the run:
- **Last succession:** None recorded

Tooltip:
- **Last succession** — The most recent change of ruler in this house.

### Expand/collapse microcopy (within the card)
- **Show household details**
- **Hide household details**

Household details (expanded) section labels:
- **Family**
- **House Log**

## Decisions screen copy (unchanged patterns; included for consistency)

Title:
- **Your Decisions**

Energy banners:
- **Low Energy** — You have {energy_remaining} left.
- **No Energy Remaining** — Review details, then end the turn.

Confirmation modal (standard):
- Title: **Confirm decision**
- Body: **Spend {energy_cost} Energy to apply this change?**
- Buttons: **Confirm** / **Cancel**

Validation:
- **Not enough Energy.** You need {needed} but have {remaining}.
- **Invalid allocation.** Total assigned exceeds available.
- **Nothing changed.** Adjust at least one value to confirm.

Toasts:
- **Decision saved.**
- **Decision reverted.**
- **Allocation updated.**

## Error states (migration / missing registry)

If v0.2.1 save data is missing registries:
- Title: **Incompatible save**
- Body: **This run can’t be loaded in v0.2.1.** *(Optional: “Start a new run.”)*

If registry exists but a referenced person is missing:
- **Missing person record.** Some household details may not display.
