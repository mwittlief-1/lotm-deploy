# LoM UX Copy Sheet — v0.2.7 (Court OS Foundations — Legibility)

**Scope:** Binding UI copy + microcopy templates only. **No new mechanics.**  
**Constraints:** clarity > vibe; one-line “why”; deterministic ordering; no invented probabilities.  
**Out of scope:** Map/Art (v0.3 lane).

---

## 1) Diff Ledger (top of Turn Report)

### Panel title
- **Diff Ledger**

Optional helper (1 line, space permitting):
- **Biggest changes from the last 3 years, with a one-line why.**

### Standard line labels (exact)
- **Food**
- **Coin**
- **Population**
- **Unrest**
- **Relations** *(used on relationship delta lines; see below)*

### Value formats (templates)

**Food (net + stored)**
- Primary: **Food: {food_delta_signed} bushels**
- Secondary (same line or subtext): **Stores: {food_stores}**
  - Example (single line): **Food: +120 bushels · Stores: 540**
  - If no unit is used elsewhere in UI, omit “bushels” consistently.

**Coin**
- **Coin: {coin_delta_signed}**

**Population (net)**
- **Population: {pop_delta_signed}**

**Unrest**
- **Unrest: {unrest_delta_signed}**

**Key relationship deltas (A/R/T)**
- **Relations ({target_name}): A {A_delta_signed} / R {R_delta_signed} / T {T_delta_signed}**
  - Target name should be a stable display name (house/person) already known to the player.

> Relationship lines: show top **N=3** movers by abs(|ΔA|+|ΔR|+|ΔT|). Do not add any “meaningful threshold” copy.

---

### “Why” attribution line (one line max)

Label (if you show a prefix):
- **Why:** {why_text}

Preferred display (no prefix if space tight):
- {why_text}

**Single-source example templates (placeholders; do not invent content):**
- Decision: **Labor reassigned.**
- Event: **Poor harvest.**
- System pressure: **Obligations due.**
- Prospect: **Marriage arranged.**

**Multiple-cause fallback (one line, neutral)**
Use when a single “why” cannot be attributed without losing truthfulness:
- **Multiple causes this turn.**

Optional (still one line) if you can safely name categories without implying details:
- **Multiple causes (decision + event).**

---

### Source tags (must be exact strings)
Render as a chip/tag aligned right (or after the why).

Allowed tags (exact):
- `decision`
- `event`
- `system_pressure`
- `prospect`

**Tag selection rule (copy-only guidance):**
- Use the highest-priority contributing category: **decision → event → system_pressure → prospect**.
- If multiple causes exist, keep the tag as the highest-priority category present and use the “Multiple causes…” why line.

---

### “Multiple causes” + tag example (combined)
- **Food: -80 · Stores: 210**  
  **Multiple causes this turn.** `event`

## 3) Council Agenda panel (3–5 items per turn)

### Panel title
- **Council Agenda**

Optional helper (1 line):

### Agenda item template (binding structure)
Each item shows:
- **{item_title}**
- {one_line_context}
- CTA: {cta_label}

**One-line context rule:**
- One sentence fragment; answer “why now” without explaining mechanics.

## Deterministic ordering (copy support)
Where the UI renders lists (Diff Ledger lines, Agenda items), ordering must be deterministic:
- Sort by score/impact (abs(delta) or deterministic rank)
- Tie-break by a stable key (id/name) so it never jitters between runs
