**Last Updated:** 2026-02-15  
**Scope:** Prospects panel/section (Marriage / Grant / Inheritance Claim-only) + relevance filtering (presentation-only)  
**Guardrails:** Copy-only. No new mechanics. All strings must tolerate missing fields (conditional omission).

## 1) Prospects section (Turn Report)

### Section title
- **Prospects**

### 1-line helper (optional, shown under title)
- **Time-limited opportunities from your network.**

### Empty states
Choose the most specific state available.

- If no prospects generated (or none present):
  - **No prospects this turn.**

- If prospects exist but none are shown (shown list empty) and hidden_count > 0:
  - **No prospects shown this turn.**
  - Helper (required): **Some opportunities are not currently relevant or actionable. You may learn of more as your network changes.**

- If shown list empty and hidden_count = 0:
  - **No prospects available yet.**

### Shown vs hidden summary (compact line; shown when hidden_count > 0)
- **Showing {shown_count} of {total_count}. Hidden: {hidden_count}.**

Tooltip (on “Hidden” label):
- **Some opportunities are not currently relevant or actionable. You may learn of more as your network changes.**

## 2) Prospect card: shared structure

> **Rule:** Omit any row when its underlying data is missing.

### Card header
- Type badge/label (one of):
  - **Marriage**
  - **Grant**
  - **Inheritance claim**

- Parties line (choose the clearest version your UI supports):
  - **From:** House {from_house_name}
  - **To:** Your House

If UI supports a single-line header:
- **House {from_house_name} → Your House**

### Subject line (optional)
- Marriage only (preferred): **Subject:** {person_name}

### Summary (required)
- Show `summary` verbatim (1 line). No extra punctuation.

### Requirements block
Label:
- **Requirements**

Content:
- Render each `requirements[].text` as a bullet.

Empty/omitted:
- If no requirements: omit the entire block.

Tooltip (on label):
- **Conditions that must be true to accept this prospect.**

### Costs block
Label:
- **Costs**

Display order (omit missing):
- **Coin:** {coin}
- **Energy:** {energy}
- **Bushels:** {bushels}

Tooltip (on label):
- **Resources spent if you accept.**

### Expected effects block (optional)
Label:
- **Expected effects**

Content rules (short, list format; omit missing):
- **Coin:** {coin_delta}
- **Relationships:** {relationship_delta_summary}
- **Flags:** {flags_set_summary}

Uncertainty label (optional; shown if present)
- **Confidence:** Known / Likely / Possible

Tooltip (on “Confidence”):
- **How certain the outcome is.**

### Expiry line (required)
- If `expires_turn` == current turn:
  - **Expires this turn.**
- Else:
  - **Expires end of Turn {expires_turn}.**

Tooltip (on “Expires”):
- **After expiry, this opportunity will no longer be actionable.**

### Actions
Buttons:
- **Accept**
- **Reject**

# v0.2.3.2 Patch Addendum — People-First Correctness + Clarity (copy-only)

> Use these templates wherever the underlying data exists. Omit lines when fields are unavailable.

## A) Widow/Widower vs Deceased status (correct subject)

### Status badges (person card / household row)
- **Alive**
- **Deceased**
- **Widow** *(surviving spouse; female)*
- **Widower** *(surviving spouse; male)*
- Fallback if sex/label unknown: **Widowed** *(surviving spouse only)*

**Rule (must):**
- The **deceased** spouse is labeled **Deceased**.
- The **surviving** spouse is labeled **Widow/Widower/Widowed**.

### Spouse line (ruler card / household header)
- If spouse alive and partner dead:
  - **Spouse:** {spouse_name} — **{Widow|Widower|Widowed}**
- If spouse is deceased:
  - **Spouse:** {spouse_name} — **Deceased**
- If no spouse:
  - **Spouse:** None

### Life log lines (consistent with v0.2.2 templates)
- Death: **Death in the household: {deceased_name}**
- Widowhood: **{surviving_spouse_name} is now a {Widow|Widower|Widowed}.**

*(Do not emit “widowed” for the deceased person.)*
